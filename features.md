# 기능 인덱스

이 문서는 기능별 산출물의 위치와 상태를 추적한다. 오케스트레이터가 Phase 4(보고) 시 자동 업데이트한다.

## 상태 표기

- **후보**: MVP 후보로 식별된 상태 (기획 착수 전)
- **기획 중**: PM이 기능 명세 작성 중
- **디자인 중**: Designer가 화면 설계 중
- **구현 중**: Developer가 구현 중
- **QA 중**: QA가 테스트 중
- **완료**: 모든 역할 산출물 완료

## MVP 후보 기능

라이너 서비스 분석(노션 기능편/UI편/기술편)과 프로젝트 핵심 가치(통합성·실시간 UX·출처 신뢰성·속도감)를 기반으로 도출한 초안. 기획 단계에서 재검토·축소·재배열 가능.

### 공통 / 프레임 레이어

| 기능 | 상태 | 설명 |
|---|---|---|
| 통합 앱 셸 (사이드바 + 중앙 패널) | 후보 | 260px 접기 가능 사이드바, 서비스 간 전환, 계정·앱 다운로드 모달. 라이너 원본 UI편 참조 |
| 인증 (NextAuth) | 후보 | 1인 로컬 사용이지만 세션·사용자 ID 기반 설계를 위해 필수. NextAuth 심층 기능 학습 목적 |
| 최근 기록 / 서비스 간 탐색 | 후보 | 사이드바 최근 기록, 서비스 링크. Scholar는 기록 제외 |

### Liner (검색·수집)

| 기능 | 상태 | 설명 |
|---|---|---|
| AI 검색 대화 (SSE 스트리밍) | 후보 | 질문 → LLM 응답 스트리밍. **Anthropic 공식 SDK 저수준 구현이 이 프로젝트 학습의 핵심** |
| 출처·인용 배지 | 후보 | 응답 내 `[n]` 형태 인용 표기, 클릭 시 원문/근거 표시 |
| 소스 컬렉션 저장 (URL/텍스트/파일) | 후보 | 사용자가 수집한 자료 저장. PDF/DOCX/HWP 파싱은 최소 MVP에서는 제외 검토 (범위 축소 후보) |
| 컬렉션을 다음 대화의 컨텍스트로 재사용 | 후보 | 컬렉션 선택 시 해당 내용이 메시지 컨텍스트로 주입. Anthropic prompt caching 적극 활용 지점 |

### Liner Write (문서 작성)

| 기능 | 상태 | 설명 |
|---|---|---|
| TipTap 기반 WYSIWYG 에디터 | 후보 | 기본 블록 타입, 자동 저장 |
| AI 수정 제안 Before/After 비교 | 후보 | 선택 영역을 AI에 보내 수정안 수신, 취소선/하이라이트로 diff 표시, 선택 적용 |
| 인용 찾기 (근거 검증) | 후보 | 본문 일부 선택 → Liner 검색으로 근거 자료 탐색 |

### Liner Scholar (프로젝트 워크스페이스)

| 기능 | 상태 | 설명 |
|---|---|---|
| 3패널 워크스페이스 (파일/편집/채팅) | 후보 | 최상단 토글로 각 패널 표시 제어. "프로젝트" 단위 컨텍스트 |
| 프로젝트 단위 파일 관리 | 후보 | 프로젝트에 파일 업로드·연결 |
| 프로젝트 컨텍스트 기반 채팅 | 후보 | 현재 프로젝트의 파일·편집 문서를 컨텍스트로 AI 대화 |

### 통합성 강화 기능 (이 프로젝트의 차별화 지점)

원본 라이너의 약점("서비스 간 컨텍스트 분리")을 직접 개선하는 기능들. **문제 2(프로덕트 관점)의 실체**.

| 기능 | 상태 | 설명 |
|---|---|---|
| 컬렉션 → Write 문서로 끌어오기 | 후보 | Liner 검색 결과·컬렉션을 Write 에디터에 인용 삽입 |
| Write 문서 → Scholar 프로젝트 연결 | 후보 | 작성 중인 문서를 Scholar 프로젝트 파일로 승격 |
| 서비스 간 컨텍스트 이동 시 대화 기록 보존 | 후보 | 서비스를 전환해도 직전 대화가 "관련 참고"로 이어짐 |
