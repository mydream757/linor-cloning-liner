# 학습 메모 (Architecture Notes)

이 디렉터리는 **"결정"이 아닌 원리·트레이드오프·배경 지식**을 담는다. ADR(`../decisions/`)이 *"우리가 X를 골랐다"*인 반면, notes는 *"X는 Y라는 이유로 동작한다. 이걸 모르면 같은 실수를 반복할 수 있다"*에 가깝다.

## 무엇을 쓸까

- 프레임워크·라이브러리의 동작 원리 중 **당장 작업에 직접 영향을 준** 것
- 두 가지 방식의 **트레이드오프** (우리가 한쪽을 고른 이유는 ADR에, *"둘 다의 특성"*은 여기에)
- 디버깅하며 알게 된 **숨은 제약** (예: Next.js 16 `cookies()`는 렌더링 중 set 불가)
- 이 프로젝트의 **도메인 모델이 특정 기술과 만나는 지점**

## 무엇을 쓰지 말까

- 결정 사항 → `../decisions/` (ADR)
- 기능 스펙 → `../../plan/features/`
- 구현 디테일 (변수명·파일 경로 등) → 코드와 커밋 메시지
- 외부 문서로 충분히 설명되는 일반론 (Next.js 공식 문서에 이미 있는 것)

## 파일 이름 규칙

`주제-짧은-설명.md` — 소문자·하이픈. 예: `nextjs-cookies-and-dynamic-rendering.md`, `prisma-cuid-vs-cuid2.md`.

## 이 디렉터리가 유용해지는 기준

나중에 같은 함정에 빠질 뻔할 때 **여기를 먼저 뒤진다**면 유용한 것. 그게 아니라면 쓰지 않아도 됨. 양이 아니라 회고 효과가 측정 기준.
