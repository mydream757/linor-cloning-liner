# Liner 뷰 — Chat 기반 AI 대화 (SSE 스트리밍) + 출처·인용 배지 — 화면 설계

> 참조
> - 기능 명세: [`plan/features/3-liner.md`](../../plan/features/3-liner.md)
> - 전역 토큰: [`../design-tokens.md`](../design-tokens.md)
> - 실측·스크린샷: [`../references/3-liner/`](../references/3-liner/)

이 문서는 기능 3의 **화면 구조·상태·인터랙션·토큰 매핑**을 정의한다. 값(픽셀·컬러)은 [`design-tokens.md`](../design-tokens.md)와 [`references/3-liner/measurements.md`](../references/3-liner/measurements.md)의 단일 진실 소스를 따르며, 여기서는 **토큰 이름과 구조 결정**만 담는다.

## 1. 사용자 흐름

```
[/p/[projectId]/liner 진입]
    │
    ├── Chat이 0개 (빈 상태)
    │       └─→ 빈 상태 화면 (CTA + 입력카드)
    │             └─→ 메시지 입력 + 전송
    │                   └─→ 새 Chat 자동 생성 + 사용자 메시지 표시
    │                         └─→ AI 응답 SSE 스트리밍 시작
    │                               ├─→ [정상 완료] 응답 확정 + 출처 배지 렌더
    │                               ├─→ [중단] Stop 클릭 → 수신분 보존
    │                               └─→ [에러] 에러 메시지 + "다시 시도"
    │
    ├── Chat이 1개 이상 (마지막 Chat 로드)
    │       └─→ 대화 화면 (메시지 목록 + 입력카드)
    │             ├─→ 메시지 전송 → 스트리밍 (위와 동일)
    │             ├─→ Chat 목록에서 다른 Chat 선택 → 해당 대화 로드
    │             ├─→ "새 대화" 클릭 → 새 Chat 생성 → 빈 대화로 전환
    │             └─→ Chat 삭제 → 확인 → 목록에서 제거
    │
    └── 출처 배지 인터랙션
            └─→ "N개의 출처" 버튼 클릭
                  └─→ 우측 출처 패널 슬라이드-인
                        └─→ 출처 카드 목록 확인 / ✕ 닫기
```

### 흐름 설명 (정상 경로)

| 단계 | 화면 | 사용자 행동 | 시스템 반응 |
|---|---|---|---|
| 1 | `/p/[id]/liner` (빈 상태) | 페이지 진입 | Chat 0개 → 빈 상태 렌더 (CTA 텍스트 + 입력카드) |
| 2 | 빈 상태 | 입력카드에 메시지 작성 + Enter (또는 전송 버튼) | 새 Chat 자동 생성. 사용자 메시지 표시. SSE 스트리밍 시작 |
| 3 | 대화 화면 | 스트리밍 관찰 | AI 응답이 글자 단위로 실시간 표시. 입력카드의 전송 버튼이 중단 버튼으로 교체 |
| 4 | 대화 화면 | 스트리밍 완료 대기 | 응답 확정. 출처 배지 `[n]` 인라인 렌더. 중단 버튼이 전송 버튼으로 복귀. 입력 가능 |
| 5 | 대화 화면 | "N개의 출처" 버튼 클릭 | 우측 출처 패널(399px) 슬라이드-인. 출처 카드 목록 표시 |
| 6 | 출처 패널 열림 | ✕ 클릭 | 출처 패널 닫힘. 메인 대화로 복귀 |
| 7 | 사이드바 Chat 목록 | 다른 Chat 클릭 | 해당 Chat의 메시지 로드. 이전 대화 맥락 표시 |
| 8 | 사이드바 Chat 목록 | "새 대화" 클릭 | 새 Chat 생성. 빈 대화 상태로 전환 |

### 예외·에러 흐름

| 조건 | 처리 방식 |
|---|---|
| 빈 메시지 전송 시도 (공백만 또는 빈 문자열) | 전송 버튼 비활성 상태 유지. 전송 차단 (클라이언트 검증) |
| 네트워크 오류로 SSE 연결 끊김 | 스트리밍 중단. 어시스턴트 메시지 하단에 에러 인라인 표시 + "다시 시도" 버튼 |
| Anthropic API 에러 (인증 실패) | 어시스턴트 메시지 영역에 "API 키가 유효하지 않습니다" 에러 표시 |
| Anthropic API 에러 (rate limit) | "요청 한도에 도달했습니다. 잠시 후 다시 시도해주세요." 에러 표시 + "다시 시도" 버튼 |
| Anthropic API 에러 (서버 오류) | "AI 서비스에 일시적 문제가 발생했습니다." + "다시 시도" 버튼 |
| 스트리밍 중 브라우저 이탈/닫기 | 서버 측 스트림 정리. 다음 접속 시 마지막 저장 시점까지의 메시지만 표시 |
| 스트리밍 중 다른 Chat으로 전환 시도 | 현재 스트리밍 abort 후 전환 (수신분 보존) |
| 스트리밍 중 새 메시지 전송 시도 | 입력카드 비활성 — 전송 불가 (중단 버튼만 표시) |
| Chat 삭제 중 서버 에러 | 삭제 confirm 다이얼로그 안에 인라인 에러 메시지. 목록 상태 변경 없음 |

**구성 요소**:

| 영역 | 요소 | 토큰 | 기능 고유 치수 | 비고 |
|---|---|---|---|---|
| 메인 패널 | 배경 | `color-bg-primary` | — | 기능 1과 동일 |

| 입력카드 | 모드 라벨 | `color-text-primary` + `text-caption` | — | "리노 리서치" (원본의 "라이너 리서치" 대응. MVP에서 Liner 드롭다운은 제외하므로 고정 라벨) |

| 사이드바 | "새 대화" 버튼 | 기존 사이드바 패턴 | w 244px, h 32px | 사이드바 "최근 기록" 섹션 상단 |
| 사이드바 | "최근 기록" 섹션 | `color-text-secondary` + `text-caption` | — | Chat 목록의 섹션 헤더 |

### 2-3. 스트리밍 중 상태

```
┌──────────────────────────────────────────────────────────────────────────┐
│                            상단 헤더 (48px)                                │
├──────────────┬───────────────────────────────────────────────────────────┤
│              │                                                           │
│   사이드바     │     사용자 메시지 ──────────────────┐                       │
│              │                      │ 질문 텍스트  │                       │
│              │                      └────────────┘                       │
│              │                                                           │
│              │     어시스턴트 응답 (스트리밍 중)                               │
│              │     텍스트가 실시간으로 한 글자씩 나타남...                       │
│              │     ▍ (커서 — 선택적)                                        │
│              │                                                           │
│              │     ┌──────────────────────────────────────┐              │
│              │     │  무엇이든 물어보세요           [■ 중단]  │              │
│              │     └──────────────────────────────────────┘              │
└──────────────┴───────────────────────────────────────────────────────────┘
```

**스트리밍 중 변경사항**:

| 요소 | 변경 | 비고 |
|---|---|---|
| 전송 버튼 → 중단 버튼 | 위치 동일 (입력카드 우하단). 40x40px, `radius-full`. 흰색 ■ 사각형 아이콘 | 클릭 시 SSE 연결 종료 |
| 입력 필드 | 비활성 (입력 불가) | 스트리밍 완료 전 추가 메시지 전송 차단 |
| 어시스턴트 메시지 | 텍스트가 실시간으로 추가됨 | 별도 타이핑 인디케이터 없음 — 텍스트 자체가 인디케이터 |
| 스크롤 | 자동 스크롤 (새 텍스트 따라감) | 사용자가 수동 스크롤 업하면 자동 스크롤 중단 |
| 응답 액션바 | 미표시 | 스트리밍 완료 후에만 표시 |

### 2-4. 에러 상태

에러는 어시스턴트 메시지 영역에 인라인으로 표시한다. 별도 토스트나 모달을 사용하지 않는다.

```
┌─ 메시지 영역 ──────────────────────────────────────────┐
│                                                        │
│                     사용자 메시지 버블                    │
│                                                        │
│  ┌────────────────────────────────────────────────┐    │
│  │  ⚠ 네트워크 오류가 발생했습니다.                    │    │
│  │                                                │    │
│  │  [ 다시 시도 ]                                   │    │
│  └────────────────────────────────────────────────┘    │
│                                                        │
└────────────────────────────────────────────────────────┘
```

**구성 요소 — 에러 메시지**:

| 속성 | 토큰/값 | 비고 |
|---|---|---|
| 컨테이너 | 없음 (어시스턴트 메시지 위치에 인라인) | — |
| 아이콘 | ⚠ 또는 SVG 경고 아이콘 | `color-text-secondary` |
| 에러 텍스트 | `color-text-secondary` + `text-body` | 에러 유형별 구체적 문구 (흐름 표 참조) |
| "다시 시도" 버튼 | `color-text-primary` + `color-border-subtle` border | 클릭 시 마지막 사용자 메시지로 재요청 |

## 5. 디자인 결정 및 근거

### Q3 해소: Chat 목록 UI 위치

**결정: 사이드바 내 "최근 기록" 섹션으로 배치한다.**

| 대안 | 장점 | 단점 | 선택 여부 |
|---|---|---|---|
| **사이드바 "최근 기록" 섹션** | 기존 사이드바 구조(Project 목록) 아래 자연스럽게 확장. 원본 Liner가 이 패턴 사용 (스크린샷 06 확인). 뷰 전환 없이 Project와 Chat을 동시에 볼 수 있음 | 사이드바 260px 폭에 두 섹션이 들어가 공간 밀도 증가 | **채택** |
| Liner 뷰 메인 패널 내 좌측 서브패널 | 메인 패널에서 Chat과 대화를 동시에 볼 수 있음 | 메인 패널의 대화 영역이 좁아짐. 기능 6(Scholar 뷰)의 3패널 레이아웃과 충돌 가능. 사이드바가 이미 있는데 메인에 또 리스트를 두면 시각적 중복 | 기각 |

**근거**: 원본 Liner의 Chat 목록이 사이드바의 "최근 기록" 섹션에 위치한다 (스크린샷 06 확인). 기능 1에서 설계한 사이드바는 Project 목록만 있어 하단 공간이 여유롭다. Chat 목록은 Project 하위 맥락이므로 현재 선택된 Project의 Chat만 보여주면 되고, 사이드바에 자연스럽게 들어간다. 또한 기능 6(Scholar 뷰)에서 메인 패널이 3패널로 분할될 때, 메인 내 Chat 리스트는 공간 충돌을 일으킨다.

**라이너 뷰(Liner) 활성 시에만 Chat 목록을 표시한다.** Write/Scholar 뷰에서는 "최근 기록" 섹션을 숨긴다. 이유: Chat 목록은 Liner 뷰의 맥락 데이터이며, 모든 뷰에서 노출하면 사이드바 공간 경합과 인지 부하가 커진다. Scholar 뷰에서 Chat은 메인 패널 우측에 별도로 재배치된다 (기능 6).

### Q4 해소: 출처 배지 클릭 시 UI

**결정: 우측 슬라이드-인 패널 (399px)을 사용한다. 진입점은 개별 배지가 아닌 응답 하단 "N개의 출처" 버튼이다.**

| 대안 | 장점 | 단점 | 선택 여부 |
|---|---|---|---|
| **우측 패널 (슬라이드-인)** | 원본 Liner와 동일 패턴 (스크린샷 05 확인). 다수의 출처를 스크롤 가능한 충분한 공간에 표시. 대화 맥락을 유지하면서 출처 확인 가능 | 화면 너비가 좁을 때 대화 영역 압축 | **채택** |
| 인라인 팝오버 | 컨텍스트 전환 없음 | 공간 제약으로 출처 정보 표시 어려움. 다수의 출처 표시 불가. 팝오버끼리 겹침 | 기각 |
| 하단 패널 | 출처 목록을 넓게 표시 | 대화 스크롤 영역 축소. 원본과 다른 패턴 | 기각 |

**근거**: 원본 Liner가 우측 패널 패턴을 사용하고 있고 (스크린샷 05, 399px), 이 패턴은 기능 4(Asset 관리)에서 Reference Asset 상세를 표시할 때도 재사용 가능하다. "N개의 출처" 버튼을 진입점으로 두는 이유: 원본 관찰에서 개별 인라인 배지 `[n]` 클릭에 대한 반응이 확인되지 않았고, 요약 버튼이 더 명확한 affordance를 제공한다.

**MVP 단계 제한**: 출처 패널의 탭("전체/웹문서/논문/컬렉션")은 시각적으로만 렌더하고, stub 데이터이므로 "전체" 탭만 동작한다. 필터링은 기능 4에서 실 Reference 데이터 연결 시 활성화한다.

| 에러를 **인라인 표시** (토스트 아님) | 토스트 알림 | 에러가 대화 흐름의 일부(어시스턴트 응답 실패)이므로 메시지 영역 내 인라인이 맥락상 자연스러움. "다시 시도"도 같은 위치에서 클릭 |
| 출처 패널 열릴 때 **메인 영역 너비 축소** | 메인 위에 overlay | 원본 패턴. 대화 맥락을 가리지 않으면서 출처를 확인할 수 있음 |

## 7. 구현자 참고사항

- **메시지 영역 스크롤**: `overflow-y: auto`를 메시지 리스트 컨테이너에 적용. 입력카드는 `position: sticky; bottom: 20px`으로 하단 고정. 메시지 영역 하단에 입력카드 높이만큼 padding 확보.
- **자동 스크롤 제어**: 스트리밍 중 `scrollTop + clientHeight >= scrollHeight - threshold`이면 자동 스크롤 유지. 사용자가 위로 스크롤하면 해제. `IntersectionObserver` 또는 scroll 이벤트로 구현.
- **출처 배지 파싱**: 어시스턴트 응답 `content`에서 `[n]` 패턴을 정규식으로 파싱하여 `<span>` 배지 요소로 변환. 파싱은 렌더 시점에 수행 (원본 content는 plain text 보존).
- **입력카드 자동 높이 조절**: `<textarea>`에 `rows=1` + `overflow: hidden` + JS로 `scrollHeight`에 따라 높이 동적 변경. 최대 높이는 뷰포트의 40% 또는 고정 max-height로 제한.
- **사이드바 Chat 목록 표시 조건**: 현재 뷰 segment가 `liner`일 때만 "최근 기록" 섹션을 렌더. 뷰 전환 시 섹션이 mount/unmount됨. transition은 불필요 (뷰 전환 자체가 라우트 변경).
- **출처 패널 슬라이드-인 모션**: `motion-medium` (150ms ease)으로 `transform: translateX` 또는 `width` 변경. 메인 메시지 영역은 `flex: 1`로 자동 축소.
- **중단 버튼과 전송 버튼 교체**: 같은 위치(입력카드 우하단)에서 조건부 렌더. `aria-label`을 "전송" / "중단"으로 각각 설정.
- **Chat 제목 자동 생성**: 첫 사용자 메시지의 `content.slice(0, 30)` + 30자 초과 시 "..." 추가. 서버 측에서 처리하고 사이드바 목록에 반영.
- **접근성**: 메시지 영역은 `role="log"` + `aria-live="polite"`. 출처 패널은 `role="complementary"`. 중단 버튼은 `aria-label="응답 중단"`.
- **반응형**: 이 문서의 치수(720px max-width, 399px 패널)는 데스크톱 기준. 모바일 대응은 MVP 범위 밖이나, `max-w-[720px]`과 퍼센트 padding으로 기본적 축소는 자연스러움.
